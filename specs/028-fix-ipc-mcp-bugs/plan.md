# Implementation Plan: Fix IPC Server & MCP Client Bugs

**Branch**: `028-fix-ipc-mcp-bugs` | **Date**: 2026-06-11 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/028-fix-ipc-mcp-bugs/spec.md`

---

## Summary

Fix three independent bugs in the Holmes agent's IPC server and MCP client:

1. **IPC deadlock** — `_handle_client` dispatches requests sequentially; `session.resolve`
   blocks the read-loop while waiting for `tool.approve`, causing a 120-second timeout.
   Fix: `asyncio.create_task` per-request dispatch with exception isolation.

2. **chat.send protocol violation** — `_handle_chat_send` returns `None`, so requests
   with a JSON-RPC `id` field never receive a response. Fix: send `{"ok": true}`
   before streaming begins.

3. **MCP client double-block** — `MCPManager._connect_server` exits its async context
   managers immediately after `initialize()`, closing the session before any tool calls.
   Additionally, `mcp_mgr.tools` is never injected into `build_tools()`. Fix:
   `AsyncExitStack` for connection lifetime + `extra_tools` parameter on `build_tools`.

## Fix Designs

### Fix 1: Concurrent dispatch in `_handle_client`

**File**: `agent/holmes/agent/ipc_server.py`

**Change**: Replace `await self._dispatch(request, writer)` with
`asyncio.create_task(self._dispatch_isolated(request, writer))`.

Add `_dispatch_isolated` method that wraps `_dispatch` with exception handling:

```python
async def _dispatch_isolated(
    self, request: dict[str, Any], writer: asyncio.StreamWriter
) -> None:
    try:
        await self._dispatch(request, writer)
    except Exception as e:
        logger.exception("Unhandled error dispatching request %s", request.get("method"))
        req_id = request.get("id")
        if req_id is not None:
            try:
                await self._send(writer, _error_response(req_id, -32603, str(e)))
            except Exception:
                pass
```

The read-loop never awaits the task — it returns to reading immediately.

**Why this is correct**: `asyncio.create_task` schedules the coroutine on the
event loop without blocking the caller. The `tool.approve` message can now be
read while `session.resolve` is suspended waiting for the Future.

### Fix 2: `chat.send` JSON-RPC acknowledgment

**File**: `agent/holmes/agent/ipc_server.py`

**Change**: Pass `req_id` to `_handle_chat_send` (or extract from writer context).

Cleanest approach: extract `req_id` from `params` inside the handler isn't possible
since params don't carry it. Instead, change `_dispatch` to pass `req_id` to handlers
that return `None` but still need to ack.

Simpler approach (no signature change): In `_dispatch`, after calling handlers that
return `None`, send the ack for methods that are known streaming handlers:

```python
# In _dispatch, after handler call:
if req_id is not None and result is None and method in _STREAMING_METHODS:
    await self._send(writer, _ok_response(req_id, {"ok": True}))
```

Where `_STREAMING_METHODS = {"chat.send", "skill.invoke"}`.

However, the cleanest solution (avoids magic set) is to pass `req_id` to `_handle_chat_send`:

Change `_handle_chat_send(self, params, writer)` to
`_handle_chat_send(self, params, writer, req_id=None)` and send ack at the top:

```python
async def _handle_chat_send(self, params, writer, req_id=None):
    if req_id is not None:
        await self._send(writer, _ok_response(req_id, {"ok": True}))
    # ... rest of existing code
```

And in `_dispatch`, call streaming handlers directly with req_id before the result check.

**Decision**: Use the `req_id` parameter approach for `_handle_chat_send` and
`_handle_skill_invoke` (both return `None` and stream). This is the cleanest and
most explicit — no hidden sets of method names.

### Fix 3: MCP connection lifetime with `AsyncExitStack`

**File**: `agent/holmes/agent/mcp_manager.py`

**Change**: Add `AsyncExitStack` to keep connections open:

```python
import contextlib

class MCPManager:
    def __init__(self, config: HolmesConfig) -> None:
        self._config = config
        self._tools: list[BaseTool] = []
        self._server_status: list[dict[str, Any]] = []
        self._exit_stack = contextlib.AsyncExitStack()

    async def initialize(self) -> None:
        await self._exit_stack.__aenter__()
        if not self._config.mcp_servers:
            return
        for server_cfg in self._config.mcp_servers:
            await self._connect_server(server_cfg)

    async def close(self) -> None:
        await self._exit_stack.__aexit__(None, None, None)

    async def _connect_server(self, server_cfg: MCPServerConfig) -> None:
        try:
            from mcp import ClientSession, StdioServerParameters
            from mcp.client.stdio import stdio_client

            params = StdioServerParameters(
                command=server_cfg.command,
                args=server_cfg.args,
                env=server_cfg.env or None,
            )
            read, write = await self._exit_stack.enter_async_context(
                stdio_client(params)
            )
            session = await self._exit_stack.enter_async_context(
                ClientSession(read, write)
            )
            await session.initialize()
            # ... proxy tool creation unchanged
```

Connections remain open until `await mcp_mgr.close()` is called.

    config=config,
    tools_factory=_tools_factory,
    socket_path=socket_path,

Add `await mcp_mgr.close()` in the shutdown path (after `await server.stop()`).
