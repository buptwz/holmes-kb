# Implementation Plan: Three-Phase Import Agent

**Branch**: `015-three-phase-import-agent` | **Date**: 2026-06-08 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/015-three-phase-import-agent/spec.md`

## Summary

Refactor the Holmes KB import pipeline from a single-loop `ImportAgentRunner` into a three-phase pipeline: **Reader → Extractor → Verifier**. Each phase runs in an isolated LLM call context (inspired by Claude Code's forked agent pattern). The Reader produces a `KnowledgeMap` episodic memory artifact that indexes all knowledge points and their document locations. Each Extractor processes one knowledge point with a clean context, using `read_document_range` tools to access the full source without truncation. The Verifier uses the full original source for field validation, eliminating false CLEAR operations on long documents. Backward-compatible: `ImportAgentRunner.run()` signature unchanged, all 455 existing tests pass.

## Implementation Phases

### Phase 1 — Foundation (US4 baseline)
- `knowledge_map.py`: KnowledgeMap + KnowledgePoint dataclasses with JSON serialization
- `doc_access.py`: DocumentCursor + three tool functions (`read_document_range`, `get_read_coverage`, `search_in_document`)
- `tools.py`: register doc_access tools in TOOL_DEFINITIONS/TOOL_HANDLERS
- `report.py`: add `knowledge_map` and `phase_traces` fields
- Tests: `test_knowledge_map.py`, `test_doc_access.py`

### Phase 2 — Reader Agent (US1, US4)
- `phases/reader.py`: ReaderAgent with fresh message loop, doc_access tools, diminishing returns detection
- `pipeline.py`: ThreePhaseImportPipeline skeleton that calls ReaderAgent then falls back to existing runner logic for Extractor/Verifier
- Tests: `test_reader_phase.py`

### Phase 3 — Extractor Agent (US2)
- `phases/extractor.py`: ExtractorAgent — one LLM loop per KnowledgePoint with clean context
- `pipeline.py`: serial Extractor loop, marks KP as extracted in KnowledgeMap
- Tests: `test_extractor_phase.py`

### Phase 4 — Verifier Integration + Skill (US1, US3)
- `verifier.py`: confirm full-doc ctx flow (W1-F1 patch already in place, just wire to pipeline)
- Post-verify: `_finalize_skill_generation` using fully verified content
- `runner.py`: wire `run()` to delegate to `ThreePhaseImportPipeline`
- Tests: `test_verifier_phase.py`, `test_pipeline.py`

### Phase 5 — Batch Verbose + Polish (US4, FR-012)
- `cli.py`: surface per-entry verbose trace in batch mode (L-W4 fix)
- End-to-end: import v13 test corpus (W1 large doc, T6 Chinese runbook, T3 multi-KP)
- All 455 tests pass
