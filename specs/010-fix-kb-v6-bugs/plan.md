## Key Implementation Notes

### US1 — placeholder comment fix (manager.py ~line 572)
Current: `"# No parameters defined via {{placeholder}} syntax"`
Fix: `"# No parameters defined via {placeholder} syntax"`
Also fix the example comment lines if they use f-string doubled braces that render as `{{`.

### US2 — dry-run for reject (cli.py kb_reject)
Add `@click.option("--dry-run", is_flag=True)`.
When `--dry-run` and `--stale-days` both set: print entry list with `(dry run)` suffix, return without deleting.

### US3 — CLAUDE.md doc change
Add section or note: `detect-commands` / `holmes kb skill detect-commands` should receive only the `## Resolution` section content, not full entry text.

### US4 — --type validation (cli.py)
Valid types inferred at runtime: `[d.name for d in kb_root.iterdir() if d.is_dir() and not d.name.startswith('.') and d.name not in ('contributions', 'skills')]`.
If `kb_type` not in valid_types: `click.echo(f"Warning: unknown type '{kb_type}'. Valid types: {sorted(valid_types)}", err=True)`.
Apply to both `kb_search()` and `kb list` (`kb_list()`).

### US5 — pending_since_source (pending.py)
In `list_pending()`, track which branch filled `pending_since` and add `"pending_since_source"` to the dict.
- Has `pending_since` field → `"field"`
- No `pending_since`, has `created_at` → `"created_at"`
- Neither → `"mtime"`
