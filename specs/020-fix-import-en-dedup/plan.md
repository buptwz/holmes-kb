# Implementation Plan: Import Pipeline v3 Bug Fixes вЂ” English Metadata & Document-Level Dedup

**Branch**: `020-fix-import-en-dedup` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/020-fix-import-en-dedup/spec.md`

## Summary

Fix two system-level bugs found in v3 usage report:
1. **US1 (P1)**: English documents produce entries missing `language: en` and `tags` due to (a) no language injection in `DraftNormalizer` and (b) YAML repair running after normalization вЂ” so when LLM generates malformed YAML for English content, normalization is silently skipped.
2. **US2 (P2)**: Re-importing the same document still creates new entries because entry-level dedup in `write_kb_entry` relies on the LLM consistently passing `source_hash` in tool calls. Fix: add a document-level pre-check in `pipeline.py` before any LLM calls.

## Implementation Details

### US1-A: Language Injection in DraftNormalizer (`normalizer.py`)

Add **Step 3a** between existing step 3 (title enforcement) and step 4 (tag extraction):

```python
# Step 3a: Language detection (new in 020).
lang = str(meta.get("language", "") or "").strip()
if not lang:
    combined = f"{title} {body}"
    has_cjk = bool(re.search(r'[\u4e00-\u9fff]', combined))
    meta["language"] = "zh" if has_cjk else "en"
    warnings.append(f'language: injected "{meta["language"]}" (auto-detected)')
```

### US1-B: Pipeline Order Fix (`pipeline.py`)

In the KP extraction loop inside `run()`, swap the order from:
```
normalizer.normalize(draft)
_validate_and_repair_draft(draft)
```
to:
```
_validate_and_repair_draft(draft)   в†ђ repair YAML first
normalizer.normalize(repaired)      в†ђ then normalize
```

### US1-C: Fallback Tag When Auto-Extraction Yields Nothing

In `DraftNormalizer._extract_tags()`: after the token loop, if `new_tags` is still empty, inject `meta.get("category", "unknown")` as a single fallback tag.
