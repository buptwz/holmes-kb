# Implementation Plan: Multi-Provider LLM Configuration

**Branch**: `014-multi-provider-llm` | **Date**: 2026-06-08 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/014-multi-provider-llm/spec.md`

## Summary

Extend Holmes import agent to support both Anthropic native API and OpenAI-compatible API (OpenAI, Azure, Ollama, etc.) through a pluggable `LLMProvider` abstraction. Users select their provider via `holmes setup --provider <anthropic|openai>`. The tool-use loop in `runner.py` and the sub-completion calls in `tools.py` both route through the provider interface.

## Detailed Design

### 1. `HolmesConfig` extension (`config.py`)

```python
@dataclass
class HolmesConfig:
    ...
    provider: str = "anthropic"   # NEW: "anthropic" | "openai"

    @classmethod
    def from_dict(cls, data: dict) -> "HolmesConfig":
        return cls(
            ...
            provider=data.get("provider", "anthropic"),  # backward-compatible default
        )
```

### 2. Provider package (`kb/agent/provider/`)

**`base.py`**:
```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Any

@dataclass
class ToolCall:
    id: str
    name: str
    input: dict[str, Any]

class LLMProvider(ABC):
    @abstractmethod
    def complete(
        self,
        messages: list[Any],
        system: str,
        model: str,
        max_tokens: int,
        tools: list[dict],
    ) -> tuple[bool, list[ToolCall], list[Any]]:
        """One step of the tool-use loop.
        Returns: (stop, tool_calls, updated_messages)
        """

    @abstractmethod
    def simple_complete(self, messages: list[dict]) -> str:
        """Single-turn text completion without tools. Returns response text."""

    @abstractmethod
    def append_tool_results(
        self,
        messages: list[Any],
        results: list[tuple[str, str]],
    ) -> list[Any]:
        """Append tool results in provider wire format. Returns updated messages."""
```

**`anthropic_provider.py`**:
- Constructor: `anthropic.Anthropic(api_key=cfg.api_key or None, base_url=cfg.api_base_url or None)`
- `complete()`: calls `client.messages.create(model, max_tokens, system, tools, messages)`, extracts `block.type == "tool_use"` blocks, returns `stop=(stop_reason == "end_turn")`
- `simple_complete()`: calls `client.messages.create(model, max_tokens, messages)`, returns `response.content[0].text`
- `append_tool_results()`: wraps results as `{"role": "user", "content": [{"type": "tool_result", "tool_use_id": id, "content": text}]}`

**`openai_provider.py`**:
- Constructor: `openai.OpenAI(api_key=cfg.api_key or None, base_url=cfg.api_base_url or None)`
- Tool definition conversion (internal helper): maps `{"name": n, "input_schema": s}` → `{"type": "function", "function": {"name": n, "description": d, "parameters": s}}`
- `complete()`: prepends system message, calls `client.chat.completions.create()`, extracts `message.tool_calls`, returns `stop=(finish_reason in {"stop", None} and not tool_calls)`
- `simple_complete()`: calls `client.chat.completions.create()`, returns `response.choices[0].message.content`
- `append_tool_results()`: appends `{"role": "tool", "tool_call_id": id, "content": text}` for each result

**`factory.py`**:
```python
def create_provider(cfg: HolmesConfig) -> LLMProvider:
    if cfg.provider == "anthropic":
        return AnthropicProvider(cfg)
    elif cfg.provider == "openai":
        return OpenAIProvider(cfg)
    raise ValueError(f"Unknown provider: {cfg.provider!r}. Must be 'anthropic' or 'openai'.")
```

### 3. `runner.py` refactor

Replace:
```python
self._client = anthropic.Anthropic(api_key=cfg.api_key or None, base_url=cfg.api_base_url or None)
```
With:
```python
from holmes.kb.agent.provider.factory import create_provider
self._provider = create_provider(cfg)
```

Refactor the tool-use loop to use `self._provider.complete()` and `self._provider.append_tool_results()`.
Pass `self._provider` in `ctx` instead of `self._client`.

### 4. `tools.py` sub-completion update

In `compare_root_cause` and `verify_content`:
- Replace `client = ctx.get("client")` with `provider = ctx.get("provider")`
- Replace `client.messages.create(...)` + `response.content[0].text` with `provider.simple_complete([{"role": "user", "content": combined_prompt}])`

### 5. `cli.py` setup command

Add `--provider` option:
```python
@click.option(
    "--provider",
    type=click.Choice(["anthropic", "openai"], case_sensitive=False),
    default="anthropic",
    help="LLM provider: 'anthropic' (Anthropic SDK) or 'openai' (OpenAI-compatible API).",
)
```

Save `cfg.provider = provider` and persist.

Update error messages:
```python
# Before (hardcoded):
"Run 'holmes setup --api-key <ANTHROPIC_API_KEY>' (requires Anthropic key for import agent)"

# After (dynamic):
f"LLM authentication failed (provider: {cfg.provider}).\n"
f"Run 'holmes setup --provider {cfg.provider} --api-key <KEY>' to reconfigure."
```
